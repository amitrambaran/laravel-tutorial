Laravel includes many features that addresses the stigma around "messy" PHP code. We will learn how to use routes and design a clean and efficient form for the uesr to submit information.


Creating your first form

Laravel makes it extremely easy for developers to create forms. After you have installed Laravel with the necessary dependancies, go ahead and create a 'contact.blade.php' file in /resources/views.


One reason why Laravel is so popular is because it handles routing with ease. Go ahead and create controllers for your form in /routes/web.php. The first parameter is the URL used to acccess the page, and the second is the controller it will use. (Eg. /contact, uses => "Controller"). A third field can also be used to specify an alias for the controller, as shown below as 'contact.store' for better readability.


Go back to your contact.blade.php file. This is where the front end is done. Create your HTML boilerplate as usual, and then include the Bulma CDN link for CSS styling.


From here, you are now free to create your own HTML form as usual. The only difference is that you will wrap each form element with div tags, and include the respective Bulma class for that element. (eg. 'div class="field", div class="radio"', etc).

Another advantage of Laravel is that it uses Composer, a dependancy manager for PHP. It also supports Artisan commands, making this process painfully easy and seamless. Open your command line (or Git Bash on Windows) within your project folder and run the following command:

$ php artisan make:controller ContactMessageController


You will now notice that there is a ContactMessageController in /http/Controllers. This is where your controllers are stored.
